# Rust Todo Widget

一个 Win64 桌面待办小组件，使用 Rust 标准库和原生 Win32 API 编写，无第三方依赖。主界面为可自定义背景色与透明度的桌面卡片风格小组件。

## 构建

```powershell
.\build_win64.ps1
```

生成文件：

```text
.\dist\todo_widget.exe
```

## 使用

- `+`：新建待办，填写事项、选择截止日期、填写截止时间和备注。
- `完`：将当前选中的待办标记完成，并从主列表隐藏。
- `查`：按名称、开始日期和结束日期查询完成记录；日期框勾选后生效。
- `周`：立即生成从当前时间往前 7 天的工作总结。
- `设`：设置每周自动汇总日，以及大模型 API URL、模型名、API Key。
- 设置窗口支持：
  - 开机自启动开关。
  - 背景色可点击选择，也可手动输入十六进制值，例如 `#F5F7FA`。
  - 窗口透明度设置，范围 `30-255`。
  - 测试大模型是否可用
- 右键待办：标记完成、编辑、查看备注、删除。删除会从所有记录中移除。
- 鼠标滚轮：滚动待办列表。
- 拖动顶部空白区域：移动小组件，松开后自动保存位置。
- 拖动窗口四边或四角：调整小组件大小，新的宽高会自动保存。
- 主窗口不再置顶，会贴近桌面层显示，不覆盖其他应用窗口。

## 数据位置

数据保存在程序所在目录的 `data` 文件夹：

```text
.\data
```

主要文件：

- `tasks.tsv`：全部待办和完成记录。
- `settings.ini`：周报日、开机启动、背景色、透明度、窗口位置和大模型配置。
- `summaries\summary-YYYY-MM-DD.md`：生成的工作总结。

## 截止时间颜色

- 距离截止 1 天内或已逾期：红色。
- 距离截止 2 天内：黄色。
- 其他：浅色。

## 大模型周报

设置里填写兼容 Chat Completions 的接口：

```text
API URL: https://.../v1 或 https://.../v1/chat/completions
模型名: 你的模型名
API Key: 你的密钥
```

如果未配置、调用失败或超过 135 秒，程序仍会生成本地草稿；调用失败详情会写入 `data\llm-error-*.txt`。调用大模型时不会弹出命令行窗口。
